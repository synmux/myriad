# DependaGroup

## Description

DependaGroup is a command-line utility that automatically configures Dependabot dependency grouping across one or more Git repositories. It enables the "all-dependencies" grouping strategy for Dependabot, which consolidates all dependency updates into a single pull request, reducing the number of dependency-related pull requests generated by GitHub's Dependabot service.

The utility scans repositories and modifies their `.github/dependabot.yml` configuration files to add a standardised dependency grouping configuration, streamlining dependency management across multiple projects.

## Installation & Setup

DependaGroup is a Bun-based TypeScript application. To use it, ensure the following dependencies are installed:

### System Requirements

- **Bun runtime** (with source map support enabled)
- **Node.js** (fs and path modules, included in runtime)

### Project Dependencies

- `commander` – Command-line interface framework
- `js-yaml` – YAML parsing and serialisation

The utility is designed to be executed directly as a script using Bun's runtime.

## Usage

### Basic Command Structure

```bash
./dependagroup.ts [directory] [options]
```

### Examples

#### Process a Specific Repository

```bash
./dependagroup.ts /path/to/repository
```

Updates the Dependabot configuration in a specific Git repository.

#### Scan Default Directory

```bash
./dependagroup.ts
```

Automatically scans the default location (`/Users/dave/src/github.com/daveio`) and processes all Git repositories found within it.

#### Dry Run Mode

```bash
./dependagroup.ts --dry-run
./dependagroup.ts -d
```

Displays what would be changed without actually modifying any files. Useful for previewing changes before applying them.

#### Combine Options

```bash
./dependagroup.ts /path/to/repository --dry-run
```

Performs a dry run on a specific repository.

## Command-Line Arguments and Flags

### Arguments

| Argument      | Description                                                                                        | Required | Default                             |
| ------------- | -------------------------------------------------------------------------------------------------- | -------- | ----------------------------------- |
| `[directory]` | Path to a specific Git repository to process. If omitted, the utility scans the default directory. | No       | `/Users/dave/src/github.com/daveio` |

| Flag        | Short Form | Description                                                                    |
| ----------- | ---------- | ------------------------------------------------------------------------------ |

## Behaviour and Implementation Details

### Repository Discovery

When no directory argument is provided, the utility performs the following steps:

1. Scans the default directory `/Users/dave/src/github.com/daveio`
2. Identifies all subdirectories that contain a `.git` folder
3. Filters out hidden directories (those beginning with a dot)
4. Returns a list of all Git repositories found

When a specific directory is provided, it validates that:

- The directory exists on the filesystem
- The directory is a valid Git repository (contains a `.git` folder)

### Configuration Processing

For each repository, the utility:

1. Checks for the existence of `.github/dependabot.yml`
2. Reads and parses the YAML configuration using `js-yaml`
3. Validates the structure (ensures `updates` array exists)
4. Adds or overwrites a `groups` object with the following structure:
   ```yaml
   groups:
     all-dependencies:
       patterns:
         - "*"
   ```
5. Applies this grouping to all update configurations
6. Writes the updated configuration back to disk with normalised formatting

### YAML Output Formatting

The utility applies specific YAML serialisation options to ensure consistent output:

- **Line width**: Unlimited (no automatic line wrapping)
- **References**: Disabled (no reference tags used)
- **Quote style**: Double quotes for strings

### Dry Run Mode

In dry run mode, the utility:

- Does not call `processRepository()` on each repository
- Only checks whether `.github/dependabot.yml` exists
- Reports which repositories would be processed
- Displays a preview message without making any file modifications
- Still provides a summary of repositories and Dependabot configurations found

### Error Handling

The utility implements robust error handling:

- **File I/O errors**: Catches and reports issues reading/writing files with descriptive messages
- **Invalid YAML**: Validates configuration structure and reports invalid configurations
- **Directory errors**: Reports when directories cannot be read or do not exist
- **Process errors**: Captures and logs errors during repository processing

### Output and Logging

The utility provides formatted console output using emoji indicators:

| Indicator | Meaning                                   |
| --------- | ----------------------------------------- |
| 📝        | Processing a repository                   |
| ✅        | Successfully updated a configuration      |
| ℹ️        | No changes were needed                    |
| ⚠️        | Invalid configuration structure detected  |
| ❌        | Error occurred during processing          |
| 🔍        | Scanning for repositories or dry run mode |
| 📊        | Summary section                           |

### Summary Report

After processing, the utility displays:

- Total number of repositories scanned
- Number of Dependabot configurations found
- Number of configurations updated (not shown in dry run mode)

## Type Definitions

The utility defines the following TypeScript interfaces for type safety:

### `DependabotGroup`

```typescript
interface DependabotGroup {
  patterns: string[];
}
```

Represents a dependency group with glob patterns.

### `DependabotUpdate`

```typescript
interface DependabotUpdate {
  "package-ecosystem": string;
  directory: string;
  schedule: {
    interval: string;
  };
  groups?: {
    [key: string]: DependabotGroup;
  };
}
```

Represents a single Dependabot update configuration for a package ecosystem.

### `DependabotConfig`

```typescript
interface DependabotConfig {
  version: number;
  updates: DependabotUpdate[];
}
```

Represents the complete Dependabot configuration structure.

## Exit Codes

| Code | Condition                                                                  |
| ---- | -------------------------------------------------------------------------- |
| 0    | Successful execution                                                       |
| 1    | Directory not found, not a Git repository, or fatal error during execution |

## Notes

- The utility modifies `.github/dependabot.yml` files in place. Back up configurations before running if caution is warranted.
- The default scan directory is hardcoded to `/Users/dave/src/github.com/daveio`. Modify the source code to change this default location.
- The utility requires write permissions to the `.github/dependabot.yml` files it modifies.
- In dry run mode, the utility still requires read permissions to access repository configurations.

## Limitations

- Only processes repositories with an existing `.github/dependabot.yml` file; it does not create Dependabot configurations from scratch.
- The default scan path is user-specific and requires modification for use in different environments.
- Currently only supports the "all-dependencies" grouping strategy; custom grouping patterns are not configurable through command-line arguments.
